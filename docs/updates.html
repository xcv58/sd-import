SD Import
Install Updates Support GitHub

Updates come from Sparkle.

The app checks the GitHub Release-hosted appcast and installs signed updates through Sparkle.

Check manually

  1. Open SD Import from Applications.
  2. Choose SD Import > Check for Updates....
  3. If Sparkle says no update is available, the app is current for the public feed.

Verify on GitHub

Compare the installed app version with the latest GitHub Release. Public releases include SD-Import.dmg, SD-Import.zip, appcast.xml, and release notes.

Latest GitHub Release

If a release breaks imports.

The rollback path is to mark the previous known-good GitHub Release as latest so Sparkle serves its appcast again, then publish a hotfix with a higher build number for anyone already affected.

Do not reuse or decrement Sparkle build numbers. Broken releases should be followed by a hotfix release with a higher APP_BUILD and clear release notes.
